# Project Rules

## Core Principles

1. Build for maintainability.
2. Build for contributors.
3. Build one feature at a time.
4. No premature optimization.
5. Every feature must have a clear purpose.

## Development Order

Backend → Data → Frontend → Mobile

## Commit Standards

feat:
fix:
docs:
refactor:
chore:

## Branch Naming

feature/<feature-name>
fix/<issue-name>
docs/<change>

## Quality Standard

Code should be understandable by a contributor seeing it for the first time.
