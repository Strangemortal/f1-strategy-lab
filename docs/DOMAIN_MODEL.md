# Domain Model

## 1. Introduction

This document defines the core entities that form the analytical foundation of F1 Strategy Lab.

The domain model represents the key concepts involved in Formula 1 race analysis and illustrates how race data, telemetry, strategy information, and recommendations relate to one another.

---

# 2. Domain Diagram

![Domain Model Diagram](./diagrams/Domain%20Model%20Diagram.png)

The domain model is centered around race analysis. Each entity contributes to the transformation of raw Formula 1 data into meaningful performance insights and strategic recommendations.

---

# 3. Core Entities

## Race

Represents a Formula 1 event.

### Responsibilities

- Stores race metadata
- Provides analytical context
- Contains participating drivers
- References the race circuit

### Examples

- Bahrain Grand Prix
- Monaco Grand Prix
- British Grand Prix

---

## Driver

Represents an individual competitor participating in a race.

### Responsibilities

- Generates telemetry data
- Completes laps
- Executes race strategy
- Receives analytical recommendations

### Examples

- Max Verstappen (VER)
- Lewis Hamilton (HAM)
- Charles Leclerc (LEC)

---

## Lap

Represents a single completed circuit by a driver.

### Responsibilities

- Stores lap performance information
- Supports driver comparison
- Provides timing metrics

### Examples

- Fastest Lap
- Qualifying Lap
- Race Lap

---

## Telemetry

Represents high-frequency vehicle performance measurements.

### Responsibilities

- Captures driver inputs
- Captures vehicle behavior
- Supports performance analysis

### Metrics

- Speed
- Throttle
- Brake
- RPM
- Gear
- DRS

---

## Track

Represents the physical race circuit.

### Responsibilities

- Defines circuit geometry
- Supports track visualization
- Supports heatmap generation

### Examples

- Bahrain International Circuit
- Silverstone Circuit
- Circuit de Monaco

---

## Strategy

Represents a driver's race execution plan.

### Responsibilities

- Tracks tyre usage
- Tracks pit stop decisions
- Supports strategic evaluation

### Examples

- One-Stop Strategy
- Two-Stop Strategy
- Alternate Tyre Strategy

---

## Stint

Represents a continuous sequence of laps completed on a single tyre compound.

### Responsibilities

- Defines tyre usage periods
- Supports strategy analysis
- Measures tyre longevity

### Examples

- Soft Tyre Stint
- Medium Tyre Stint
- Hard Tyre Stint

---

## Recommendation

Represents strategic guidance generated by the platform.

### Responsibilities

- Evaluates current race conditions
- Assesses tyre life
- Suggests pit stop timing

### Examples

- Box This Lap
- Extend Current Stint
- Pit Within Five Laps

---

# 4. Entity Relationships

The primary relationships within the domain are:

```text
Race
 ├── Driver
 ├── Lap
 └── Track

Driver
 ├── Telemetry
 ├── Strategy
 ├── Lap
 └── Recommendation

Strategy
 └── Stint

Lap
 └── Telemetry
```

These relationships define how race data is organized and analyzed throughout the platform.

---

# 5. Analytical Perspective

The domain model supports the complete analytical workflow of the platform.

```text
Race
   ↓
Driver
   ↓
Lap
   ↓
Telemetry
   ↓
Track Analysis
   ↓
Strategy Evaluation
   ↓
Stint Analysis
   ↓
Recommendation
```

This progression represents the journey from raw race data to actionable racing intelligence.

# 7. Summary

The F1 Strategy Lab domain consists of eight primary entities:

- Race
- Driver
- Lap
- Telemetry
- Track
- Strategy
- Stint
- Recommendation

Together, these entities provide the conceptual foundation for telemetry analysis, track visualization, strategy evaluation, and recommendation generation across the platform.
